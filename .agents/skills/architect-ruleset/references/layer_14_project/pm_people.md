# Seção 4 – Regras de Gestão de Pessoas e Equipes (Layer 14 - PM-PEOPLE)

---

### REGPM-005 – Composição de Equipe com Personalidades Complementares

| Campo | Conteúdo |
| :--- | :--- |
| **ID** | REGPM-005 |
| **Nome** | Composição de Equipe com Personalidades Complementares |
| **Descrição** | O agente deve recomendar que a equipe seja composta por pessoas com personalidades complementares, baseadas em três tipos principais: **(1)** Orientadas a Tarefas: motivadas pelo desafio técnico; **(2)** Orientadas a Si Mesmas: motivadas por reconhecimento e progresso na carreira; **(3)** Orientadas a Interação: motivadas pela interação com outras pessoas, facilitam a comunicação. O agente deve evitar equipes com predominância de um único tipo, pois isso pode levar a conflitos ou falta de equilíbrio. |
| **Objetivo** | Criar equipes equilibradas, com habilidades técnicas e interpessoais complementares. |
| **Motivação** | Cap. 22.3.1 – a composição da equipe afeta a coesão e o desempenho. |
| **Justificativa** | Equipes com personalidades diversas são mais resilientes e criativas. |
| **Critérios de Aplicação** | Formação de novas equipes ou recomposição de equipes existentes. |
| **Critérios de Não Aplicação** | Equipes com menos de 3 pessoas (onde a diversidade é limitada). |
| **Pré-condições** | Perfis dos candidatos disponíveis. |
| **Pós-condições** | Equipe formada com equilíbrio de perfis. |
| **Restrições** | O agente deve considerar também as habilidades técnicas e experiência. |
| **Dependências** | Nenhuma. |
| **Prioridade** | **Média** |
| **Severidade** | **Crítica** |
| **Exemplo Positivo** | "Equipe com 2 orientados a tarefas, 1 orientado a si mesmo, 2 orientados a interação." |
| **Exemplo Negativo** | "Equipe com 5 orientados a tarefas, todos competindo por reconhecimento técnico." |
| **Anti-pattern** | Ignorar a dinâmica de personalidades e focar apenas em habilidades técnicas. |
| **Métrica** | Diversidade de perfis na equipe (meta: pelo menos 2 tipos). |
| **Critérios de Auditoria** | Revisar a composição da equipe e o histórico de conflitos. |

### REGPM-006 – Motivação da Equipe com Base na Hierarquia de Maslow

| Campo | Conteúdo |
| :--- | :--- |
| **ID** | REGPM-006 |
| **Nome** | Motivação da Equipe com Base na Hierarquia de Maslow |
| **Descrição** | O agente deve recomendar ações para motivar a equipe com base na hierarquia de necessidades de Maslow: **(1)** Necessidades Fisiológicas e de Segurança: garantir salários justos, ambiente de trabalho seguro, estabilidade; **(2)** Necessidades Sociais: promover interação, eventos sociais, integração; **(3)** Necessidades de Estima: reconhecer publicamente conquistas, dar feedback positivo; **(4)** Necessidades de Autorrealização: oferecer desafios, autonomia, oportunidades de aprendizado. O agente deve adaptar as ações ao perfil predominante da equipe. |
| **Objetivo** | Manter a equipe motivada e engajada, reduzindo rotatividade e aumentando produtividade. |
| **Motivação** | Cap. 22.2.1 – a motivação é um fator crítico para o desempenho da equipe. |
| **Justificativa** | Equipes motivadas são mais produtivas, cometem menos erros e têm maior coesão. |
| **Critérios de Aplicação** | Durante todo o ciclo de vida do projeto. |
| **Critérios de Não Aplicação** | Nenhum. |
| **Pré-condições** | Conhecimento do perfil da equipe. |
| **Pós-condições** | Plano de engajamento e motivação implementado. |
| **Restrições** | As ações devem ser adaptadas à cultura organizacional. |
| **Dependências** | REGPM-005. |
| **Prioridade** | **Média** |
| **Severidade** | **Crítica** |
| **Exemplo Positivo** | "Oferecemos treinamentos, feedback regular e reconhecimento público; a moral da equipe está alta." |
| **Exemplo Negativo** | "Não nos preocupamos com motivação; as pessoas são pagas para trabalhar." |
| **Anti-pattern** | Oferecer apenas benefícios financeiros, ignorando necessidades sociais e de estima. |
| **Métrica** | Taxa de rotatividade da equipe (meta: < 10% ao ano). |
| **Critérios de Auditoria** | Realizar pesquisas de clima organizacional. |

### REGPM-007 – Coesão de Grupo com Reuniões Regulares e Alinhamento de Objetivos

| Campo | Conteúdo |
| :--- | :--- |
| **ID** | REGPM-007 |
| **Nome** | Coesão de Grupo com Reuniões Regulares e Alinhamento de Objetivos |
| **Descrição** | O agente deve promover a coesão do grupo por meio de: **(1)** Reuniões regulares (ex: daily stand-ups, retrospectivas) para alinhar expectativas e compartilhar informações; **(2)** Definição clara de objetivos comuns (ex: metas de sprint, marcos); **(3)** Eventos sociais (ex: almoços, atividades de team building) para fortalecer relacionamentos; **(4)** Transparência: compartilhar informações sobre o projeto, decisões e desafios. O agente deve garantir que todos os membros da equipe se sintam parte do grupo e tenham voz nas decisões. |
| **Objetivo** | Fortalecer a coesão da equipe, melhorando a comunicação, a confiança e o comprometimento. |
| **Motivação** | Cap. 22.3 – a coesão do grupo é um fator crítico para o sucesso do projeto. |
| **Justificativa** | Equipes coesas são mais resilientes, comunicam-se melhor e têm menos conflitos. |
| **Critérios de Aplicação** | Todo projeto com mais de 3 pessoas. |
| **Critérios de Não Aplicação** | Equipes muito pequenas (1-2 pessoas) onde a coesão é menos crítica. |
| **Pré-condições** | Equipe formada. |
| **Pós-condições** | Plano de coesão implementado. |
| **Restrições** | As atividades de team building devem ser adaptadas à cultura da equipe. |
| **Dependências** | REGPM-006. |
| **Prioridade** | **Média** |
| **Severidade** | **Crítica** |
| **Exemplo Positivo** | "Daily stand-ups diários, retrospectivas quinzenais, almoços mensais. Equipe muito coesa." |
| **Exemplo Negativo** | "Cada um trabalha no seu canto; raramente nos falamos." |
| **Anti-pattern** | Reuniões longas e improdutivas que desmotivam a equipe. |
| **Métrica** | Participação em reuniões e eventos (meta: > 90%). |
| **Critérios de Auditoria** | Verificar se há registro de reuniões e eventos de integração. |

**ID:** ARCH-RULESET-L10-PROC-GOV  
**Status:** Definitivo  
**Escopo:** Critérios para documentação obrigatória do ciclo de vida e dos fluxos do processo adotado.

### REGPROC-012 – Documentação Obrigatória do Processo (para Projetos Plan-Driven e Ágeis)

| Campo | Conteúdo |
| :--- | :--- |
| **ID** | REGPROC-012 |
| **Nome** | Documentação Obrigatória do Processo (para Projetos Plan-Driven e Ágeis) |
| **Descrição** | Independentemente do modelo de processo adotado, o agente deve garantir que o processo seja documentado em, no mínimo, três níveis: **(1)** Visão Geral (propósito, fases, entregáveis); **(2)** Detalhamento (atividades, papéis, ferramentas); **(3)** Critérios de Conclusão (como saber que uma atividade está concluída). Em projetos ágeis, a documentação deve ser leve (ex: wiki, README) mas suficiente para orientar novos membros e permitir auditoria. O agente deve evitar documentação excessiva que se torne obsoleta. |
| **Objetivo** | Garantir que o processo seja compreendido, seguido e passível de melhoria contínua. |
| **Motivação** | Cap. 2.2, Cap. 24.2 – a documentação do processo é essencial para qualidade e governança. |
| **Justificativa** | Processos não documentados são seguidos de forma inconsistente, gerando retrabalho e baixa qualidade. |
| **Critérios de Aplicação** | Todo projeto de desenvolvimento de software. |
| **Critérios de Não Aplicação** | Nenhum. |
| **Pré-condições** | Processo definido. |
| **Pós-condições** | Documento de processo aprovado e disponível. |
| **Restrições** | A documentação deve ser revisada periodicamente para refletir mudanças. |
| **Dependências** | REGDOC-001 (documentação arquitetural). |
| **Prioridade** | **Alta** |
| **Severidade** | **Crítica** |
| **Exemplo Positivo** | "Documento de processo Scrum com 10 páginas, incluindo papéis, cerimônias, e artefatos." |
| **Exemplo Negativo** | "Não temos processo documentado; cada um faz do seu jeito." |
| **Anti-pattern** | Documentar o processo em um PDF de 200 páginas que ninguém lê. |
| **Métrica** | Percentual de projetos com processo documentado. |
| **Critérios de Auditoria** | Verificar se o documento de processo existe e está atualizado. |
